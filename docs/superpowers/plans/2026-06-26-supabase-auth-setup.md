# Supabase Project & Auth Setup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create and link a Supabase project, wire the environment, and add login + register screens with an auth gate so every subsequent game system has a real user session to work with.

**Architecture:** Supabase handles auth via JWT + AsyncStorage session persistence (already configured in `src/services/supabase.ts`). Expo Router's file-based routing groups unauthenticated users under `app/(auth)/` and authenticated users under `app/(tabs)/`. The root layout listens to `onAuthStateChange` and redirects between groups — no extra state management needed.

**Tech Stack:** Expo Router 56, `@supabase/supabase-js` 2.x (already installed), React Native core components, Supabase CLI (to be installed), `src/constants/theme.ts` for styling.

> **Run this plan BEFORE** `2026-06-26-spin-economy-ships.md` — the spin/economy plan's Edge Functions and stores require an authenticated session and a linked Supabase project.

## Global Constraints

- Never hardcode Supabase URL or keys — use `EXPO_PUBLIC_*` env vars as shown in `.env.example`
- `.env` is gitignored — never commit it; `.env.example` (already committed) documents the required keys
- All auth calls go through `src/services/supabase.ts` — no direct `createClient` calls elsewhere
- TypeScript strict mode — no `any`, no `!` assertions without a comment
- Follow existing style in `src/constants/theme.ts` for all colors, spacing, font sizes, radii

- [ ] **Step 4: Update `app/_layout.tsx` with the auth gate**

  Replace the entire file:

  ```tsx
  import { useEffect, useState } from 'react';
  import { Stack, useRouter, useSegments } from 'expo-router';
  import { StatusBar } from 'expo-status-bar';
  import { GestureHandlerRootView } from 'react-native-gesture-handler';
  import type { Session } from '@supabase/supabase-js';
  import { supabase } from '@/src/services/supabase';

  export default function RootLayout() {
    const router   = useRouter();
    const segments = useSegments();
    // undefined = still loading; null = no session; Session = authenticated
    const [session, setSession] = useState<Session | null | undefined>(undefined);

    useEffect(() => {
      // Hydrate session from AsyncStorage on first load
      supabase.auth.getSession().then(({ data: { session } }) => setSession(session));

      const { data: { subscription } } = supabase.auth.onAuthStateChange((_event, session) => {
        setSession(session);
      });
      return () => subscription.unsubscribe();
    }, []);

    useEffect(() => {
      if (session === undefined) return; // still loading — don't redirect yet

      const inAuthGroup = segments[0] === '(auth)';

      if (!session && !inAuthGroup) {
        // Not signed in and not already on an auth screen → send to login
        router.replace('/(auth)/login');
      } else if (session && inAuthGroup) {
        // Signed in but still on an auth screen → send to the game
        router.replace('/(tabs)');
      }
    }, [session, segments]);

    return (
      <GestureHandlerRootView style={{ flex: 1 }}>
        <Stack screenOptions={{ headerShown: false }}>
          <Stack.Screen name="(auth)" />
          <Stack.Screen name="(tabs)" />
          <Stack.Screen name="+not-found" />
        </Stack>
        <StatusBar style="light" />
      </GestureHandlerRootView>
    );
  }
  ```
